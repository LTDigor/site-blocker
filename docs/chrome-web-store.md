# Chrome Web Store release

## First manual setup

The first Chrome Web Store item must be created in the Developer Dashboard because this repository does not yet have a store item ID.

1. Create the store item in the Chrome Web Store Developer Dashboard.
2. Upload the Chromium ZIP generated by `npm run package`.
3. Complete Store Listing, Privacy, and Distribution.
4. Set visibility to `Unlisted`.
5. Publish once manually after the listing is complete.
6. Save the resulting extension item ID as the GitHub secret `CWS_EXTENSION_ID`.

If the dashboard visibility is changed later, publish manually once with the new visibility before using the API again.

## GitHub secrets

Configure these repository secrets before pushing release tags:

- `CWS_PUBLISHER_ID`: Chrome Web Store publisher ID.
- `CWS_EXTENSION_ID`: Chrome Web Store item ID.
- `CWS_SERVICE_ACCOUNT_JSON`: Google Cloud service account JSON key.

The service account must be added to the Chrome Web Store Developer Dashboard account and the Chrome Web Store API must be enabled in its Google Cloud project.

## Release flow

1. Update `package.json` and `manifest.json` to the same SemVer value.
2. Run `npm test`.
3. Run `npm run package`.
4. Inspect `dist/site-blocker-chromium-v<version>.zip`.
5. Create and push a matching tag, for example `v1.2.0`.

The `Chrome Web Store Release` workflow validates the tag, runs tests, builds the ZIP, uploads it as a GitHub artifact, uploads it to Chrome Web Store API v2, and publishes it for review using the item’s existing visibility settings.

## Store listing defaults

Single purpose: block user-selected websites or URLs and redirect them to a local block page.

Permission justifications:

- `storage`: stores the user’s block list and optional custom block image locally.
- `declarativeNetRequest`: applies browser-level blocking rules.
- `activeTab`: reads the active tab only when the user clicks `Add current site`.
- `alarms`: refreshes blocking rules when a temporary 10-minute unblock expires.
- `http://*/*` and `https://*/*`: allows blocking user-selected websites across normal web pages.

Privacy disclosure: the extension does not collect, sell, or share remote user data. The block list and optional custom image are stored in Chrome extension local storage.
